The Challenge

Accidents kill
because help arrives late

01

Victim Cannot Call

Severe crashes often render the driver unconscious or immobile — physically unable to dial emergency services or describe their location.

02

Exact Location Unknown

Even conscious victims struggle to describe precise crash locations on highways, rural roads or unfamiliar areas — delaying ambulance dispatch.

03

The Golden Hour

Trauma medicine defines the first 60 minutes as critical. Each minute of delayed response exponentially increases fatality risk and permanent injury.

— Our Solution
  • 3-axis accelerometer detects sudden g-force deviation automatically — no human action
  • NEO-6M GPS captures precise coordinates at the moment of impact
  • SIM800 GSM sends SMS with direct Google Maps link to emergency contact
  • 10-second cooldown prevents repeated SMS floods from post-crash vibration
  • Boot-time SMS confirms system readiness — tests the full pipeline on startup
  • Fully standalone — no internet, no app, no smartphone required by victim
Working Principle

Four steps from
crash to rescue

01
📡

Continuous Monitoring

Accelerometer reads X, Y, Z axes every 500ms. Baseline calibrated at startup — steady-state values stored as reference.

02
💥

Impact Detected

Crash g-force causes axis deviation beyond ±30 units from baseline. Any single axis triggers the alert pipeline immediately.

03
🛰️

GPS Coordinates

TinyGPS++ decodes live NMEA sentences from NEO-6M. Lat/lng extracted at 6-decimal precision — encoded as Google Maps URL.

04
📱

SMS Dispatched

SIM800 executes AT command sequence — text mode, recipient, message body, Ctrl+Z. Full cycle completes in under 8 seconds.

Components

Hardware
breakdown

Each component chosen for standalone reliability — no WiFi, no cloud dependency, functional in any location with GSM network coverage.

🤖
Arduino Board
ATmega328P

Main controller running the firmware. Manages two SoftwareSerial channels and reads analog accelerometer values every 500ms.

📶
GSM Module
SIM800 / SIM900

Sends SMS via AT command protocol over SoftwareSerial (pins 5/7). Requires active SIM with SMS plan — works on any GSM network.

🛰️
GPS Module
NEO-6M · TinyGPS++

Parses NMEA satellite sentences via SoftwareSerial (pins 12/11). TinyGPS++ extracts lat/lng with 6-decimal accuracy — ~0.1m precision.

📐
Accelerometer
3-Axis Analog

Analog X/Y/Z output on pins A0–A2. Values 0–1023. Deviation of ±30 from baseline on any axis triggers crash detection logic.

📟
SIM Card
SMS Enabled

Standard SIM with active SMS pack. Emergency contact number stored in firmware — changeable in a single line of code.

⚡
Power Supply
5V · 2A Peak

Stable 5V for Arduino. GSM module demands 2A peak during SMS transmission — dedicated regulated supply prevents voltage drops.

detect_crash.ino
// Baseline set at steady-state int bX=733, bY=705, bZ=704; const int THRESHOLD = 30; bool detectCrash() { int x = analogRead(A0); int y = analogRead(A1); int z = analogRead(A2); // Check any axis deviation if (abs(x-bX) > THRESHOLD || abs(y-bY) > THRESHOLD || abs(z-bZ) > THRESHOLD) { bX=x; bY=y; bZ=z; return true; } return false; }

main_loop.ino
void loop() { readGPS(); // parse NMEA sentences if (detectCrash()) { String msg = "⚠️ Movement Detected!\n"; if (latitude != "") { msg += "Location: https://maps." + "google.com/?q=" + latitude + "," + longitude; } else { msg += "GPS not available yet."; } sendSMS(msg); delay(10000); // spam guard } delay(500); }
read_gps.ino
// TinyGPS++ parses NMEA sentences void readGPS() { while (gpsSerial.available()) { gps.encode(gpsSerial.read()); } if (gps.location.isValid()) { // 6dp ≈ 0.1m precision latitude = String( gps.location.lat(), 6); longitude = String( gps.location.lng(), 6); } }
send_sms.ino
void sendSMS(String text) { // 1. Set text mode gsmSerial.println("AT+CMGF=1"); delay(1000); // 2. Recipient number gsmSerial.println( "AT+CMGS=\"+91XXXXXXXXXX\""); delay(1000); // 3. Message body gsmSerial.print(text); delay(1000); // 4. Ctrl+Z → dispatch gsmSerial.write(26); delay(5000); }
setup.ino
// Dual SoftwareSerial channels SoftwareSerial gsmSerial(5, 7); SoftwareSerial gpsSerial(12, 11); TinyGPSPlus gps; void setup() { Serial.begin(9600); gsmSerial.begin(9600); gpsSerial.begin(9600); delay(1000); // Boot confirmation sendSMS("System is ready."); }

My Role

What
I built

01

Accelerometer Threshold Logic

Calibrated baseline X/Y/Z for steady-state. Implemented 3-axis deviation check with ±30 unit threshold — updates baseline on detection to track new position.

02

Dual SoftwareSerial Management

Managed concurrent GSM (pins 5/7) and GPS (pins 12/11) SoftwareSerial channels on a single Arduino without bus collision.

03

GPS Parsing with TinyGPS++

Integrated TinyGPS++ to continuously decode NMEA sentences. Extracts valid lat/lng at 6-decimal precision and encodes as Google Maps URL.

04

AT Command SMS Pipeline

Built complete SMS dispatch sequence — CMGF text mode, CMGS recipient, message body, ASCII 26 Ctrl+Z — with calibrated timing delays for reliability.

05

Spam Prevention + Boot Test

10-second delay prevents SMS flooding from post-crash vibration. Boot-time "System is ready" SMS validates full GSM pipeline on every startup.
